## 1. ОБЩАЯ КОНЦЕПЦИЯ

### 1.1 Философия
Сайт визуализирует невидимый бэкенд. Каждый проект — это интерактивная сцена, которая показывает "как выглядит то, что скрыто под капотом". Посетитель не читает — он взаимодействует. Это не список проектов, а путешествие через 5 миров, каждый из которых демонстрирует конкретный навык.

### 1.2 Ключевые принципы
- **Нет скроллбара** — страница не скроллится в классическом смысле. Колесо мыши / свайп перехватываются JS и управляют анимацией переходов между секциями (scroll-hijacking через `overflow: hidden` на body).
- **Каждая секция = полноэкранная сцена** — никакого "обычного сайта". Каждый экран занимает 100vh × 100vw.
- **Интерактивность везде** — курсор влияет на всё, каждый объект кликабелен или draggable.
- **Звук усиливает ощущение** — тонкие звуковые эффекты при взаимодействии (не музыка, а feedback).
- **Производительность** — 60fps обязательно. Ленивая загрузка сцен, LOD для 3D, requestAnimationFrame.

## 4. SCROLL-СИСТЕМА (SCROLL HIJACKING)

### 4.1 Механика
```
body { overflow: hidden; height: 100vh; }
```

Все секции лежат друг на друге (position: absolute или через трансформы). Видима только текущая. Wheel-события перехватываются:

```javascript
// Псевдокод логики
let currentSection = 0;
const totalSections = 7; // hero + 5 проектов + contact
let isAnimating = false;

window.addEventListener('wheel', (e) => {
    e.preventDefault();
    if (isAnimating) return;

    const direction = e.deltaY > 0 ? 1 : -1;
    const next = clamp(currentSection + direction, 0, totalSections - 1);

    if (next !== currentSection) {
        isAnimating = true;
        animateTransition(currentSection, next, () => {
            currentSection = next;
            isAnimating = false;
        });
    }
});
```

### 4.2 Переходы между секциями
Каждый переход — уникальная анимация длительностью 1.0–1.5 секунды:

- **Hero → Проект 1**: Камера "пролетает" вперёд сквозь particle field, частицы разлетаются в стороны. Текст hero растворяется с blur-эффектом.
- **Проект → Проект**: Текущая сцена "схлопывается" в точку с эффектом хроматической аберрации. Новая сцена "разворачивается" из этой точки.
- **Проект 5 → Контакты**: Все элементы мягко растворяются в aurora-градиент, из которого выплывает контактная секция.

### 4.3 Индикатор навигации
Справа по центру — вертикальная полоска точек (7 штук). Текущая секция подсвечивается aurora-градиентом и увеличена. Точки кликабельны для прямой навигации. При переходе — плавное заполнение линии между точками.

### 4.4 Мобильная адаптация
На мобильных scroll-hijacking работает через touch-события (touchstart/touchend для определения направления свайпа). Порог: минимум 50px свайпа для срабатывания.

### 5.2 3D-объект на героэкране
**Светящаяся сфера из частиц (particle sphere):**
- ~5000 частиц формируют сферу, которая "дышит" (масштаб пульсирует по синусоиде).
- Частицы соединены тонкими линиями, образуя сетку (как нейронная сеть).
- Сфера медленно вращается.
- **Реакция на курсор**: при приближении курсора ближайшие частицы отталкиваются, создавая "вмятину". После отведения курсора — пружинят обратно.
- **Цвет**: aurora-градиент (зелёный → голубой → фиолетовый) перетекает по сфере.
- Реализация: Three.js Points с кастомным vertex shader для деформации + fragment shader для цвета.

### 5.3 Фоновый эффект
**Aurora noise field** на весь экран за 3D-объектом:
- Полноэкранный plane с fragment shader.
- Simplex noise 3D, медленно анимированный по третьей координате (time).
- Цвета: тёмные вариации aurora-палитры (приглушённые, чтобы не отвлекать от контента).
- Курсор создаёт "свечение" — в районе курсора noise усиливается / становится ярче.
- Эффект присутствует на ВСЕХ секциях как фоновый слой, но меняет палитру под каждую сцену.

### 5.4 Текст "EDWARD"
- Шрифт: крупный (clamp(48px, 8vw, 120px)), bold, Inter или кастомный.
- Эффект при загрузке: typing-анимация с курсором-блинком (как в терминале), ИЛИ glitch-эффект (буквы рандомно меняются, потом "устаканиваются").
- На hover каждая буква слегка подпрыгивает (stagger-анимация через GSAP).

### 5.5 Индикатор "scroll to explore"
- Текст + стрелки вверх-вниз (↕), моноширинный шрифт.
- Пульсация opacity: 0.3 → 1.0 → 0.3 (цикл 2 секунды).
- Мягкое движение вверх-вниз (translateY ±5px).
- Исчезает при первом скролле (fade out).

### 6.1 ПРОЕКТ 1: "Bot Swarm" — Автоматизация и боты

**Концепция**: Экран-террариум с живыми ботами-агентами.

**Визуал**:
- Тёмный фон с сеткой (grid), как мониторинг-дашборд.
- 8–12 ботов-агентов — маленькие геометрические существа (octahedron/icosahedron с "глазами" — два светящихся кружка).
- Каждый бот имеет свой цвет из aurora-палитры и свою "профессию".

**Типы ботов и их поведение**:
- **Parser Bot** (зелёный): "ест" строки текста, которые падают сверху. При поедании — текст рассыпается на символы.
- **Messenger Bot** (голубой): периодически отправляет конвертики (маленькие квадраты), которые летят к краю экрана. Видны мини-уведомления "✓ Sent".
- **Monitor Bot** (фиолетовый): вращает "радар" (вращающаяся линия), при обнаружении аномалии — мигает красным.
- **Scheduler Bot** (розовый): имеет над собой мини-таймер, периодически "запускает" других ботов.
- **Database Bot** (жёлтый): складывает данные в визуальную "стопку" блоков.

**Интерактивность**:
- **Клик на бота** → рядом с ботом открывается мини-терминал (DOM-элемент, стилизованный как консоль) с логом его действий в реальном времени. Текст набирается с typing-эффектом.
  ```
  [12:03:41] Parsing RSS feed...
  [12:03:42] Found 23 new entries
  [12:03:42] Filtering by keywords...
  [12:03:43] 3 entries matched → sending to queue
  ```
- **Drag-and-drop**: можно перетаскивать ботов, они сопротивляются (пружинная физика), потом возвращаются на место.
- **Кнопка "Deploy New Bot"**: появляется новый бот с анимацией "сборки" (частицы собираются в форму).
- **Hover на бота**: бот увеличивается, вокруг него появляется пунктирная обводка и label с именем.

**Звук**: тихие клики при действиях ботов, мягкий "blip" при отправке сообщений, механический звук при деплое нового бота.

**Теги**: `Python` `asyncio` `Telegram API` `Celery` `Redis` `Cron`

### 6.2 ПРОЕКТ 2: "API Nexus" — Интеграции и API

**Концепция**: 3D-карта сервисов, соединённых data-трубами.

**Визуал**:
- Изометрический/3D вид (камера под углом ~30°).
- 8–10 узлов-сервисов расположены в пространстве. Каждый узел — икосфера или куб с логотипом/иконкой сервиса внутри (Telegram, Stripe, OpenAI, GitHub, AWS S3, PostgreSQL, Redis, Twilio).
- Между узлами — светящиеся "трубы" (линии с glow-эффектом), по которым периодически летят пакеты данных (маленькие светящиеся кубики).
- Пакеты несут "данные" — при hover на летящий пакет видно мини-JSON.

**Интерактивность**:
- **Кнопка "Send Request"**: Запускает визуальный запрос. Появляется стартовый пакет, который проходит цепочку:
  ```
  User Input → Auth Gateway → API Endpoint → Transform → Database → Response
  ```
  Каждый этап подсвечивается при прохождении пакета. На каждом этапе пакет визуально трансформируется (меняет цвет/форму).
- **Клик на узел**: Узел "раскрывается" — видна мини-архитектурная схема внутренностей (endpoints, middleware). DOM-overlay с информацией.
- **"Break" режим**: Кнопка для имитации ошибки — один узел "ломается" (красное свечение, искры). Видно как система обрабатывает ошибку: retry-стрелки, fallback-маршрут подсвечивается альтернативным цветом, error-лог в мини-терминале.
- **Drag узлы**: Можно перетаскивать узлы, трубы пружинят и перестраиваются.

**Звук**: электронный "whoosh" при пролёте пакета, мягкий "ping" при достижении узла, тревожный тон при ошибке.

**Теги**: `REST` `WebSocket` `GraphQL` `OAuth` `Rate Limiting` `Retry Logic`

### 6.3 ПРОЕКТ 3: "Cipher Room" — Шифрование и безопасность

**Концепция**: Визуальная шифровальная машина.

**Визуал**:
- Тёмная "комната" с центральной "машиной". Машина — набор визуальных шестерёнок, роторов, проводов (стилизация под Enigma, но футуристичная).
- Слева — поле ввода (стилизованное как terminal input).
- Справа — поле вывода (зашифрованный текст с glow-эффектом).
- Между ними — визуальный pipeline обработки.

**Интерактивность**:
- **Ввод текста**: Посетитель печатает текст. Каждая буква визуально "влетает" в машину:
  1. Буква превращается в бинарный код (анимация разлёта на 0 и 1)
  2. Биты пролетают через "шестерёнки" алгоритма (шестерёнки вращаются)
  3. На выходе — зашифрованные байты собираются в hex-строку
- **Выбор алгоритма**: Переключатели AES-256, RSA, SHA-256, bcrypt. При переключении машина визуально перестраивается (шестерёнки меняют расположение, появляются/исчезают элементы) с анимацией морфинга.
- **Режим "Brute Force"**: Кнопка запускает визуальную атаку:
  - Счётчик комбинаций, который бешено крутится
  - Прогресс-бар, который еле ползёт (для AES-256) или движется быстрее (для слабых ключей)
  - Визуальные "удары" по замку
  - Текст с оценкой времени: "~2^256 years remaining"
- **Hover на элементы машины**: Пояснения, что делает каждая часть (S-Box, Key Schedule, Rounds и т.д.).

**Звук**: механические клики шестерёнок при обработке, печатная машинка при вводе, тревожные звуки при brute force.

**Теги**: `Cryptography` `AES` `RSA` `Hashing` `Security` `Python hashlib`

### 6.4 ПРОЕКТ 4: "Data Flow" — Базы данных и пайплайны

**Концепция**: Визуализация потока данных от хаоса к структуре.

**Визуал**:
- Вертикальный поток (сверху вниз).
- **Сверху**: "источник" — из "трубы" льются сырые данные: хаотичные символы, разноцветные частицы разных размеров, "грязные" строки.
- **Середина**: этапы обработки — визуальные "фильтры" (горизонтальные полосы с иконками):
  1. **Validation** — красные (невалидные) частицы отсеиваются в сторону, зелёные проходят.
  2. **Transform** — частицы меняют форму (круг → квадрат, смена цвета).
  3. **Normalize** — все частицы становятся одного размера.
  4. **Index** — частицы выстраиваются в упорядоченные ряды.
- **Внизу**: "база данных" — визуально кристаллическая решётка, где данные аккуратно уложены. Решётка мягко светится.

**Интерактивность**:
- **SQL-консоль**: Внизу экрана — поле ввода. Можно написать SQL-подобный запрос:
  ```sql
  SELECT * WHERE color = 'green'
  ```
  "Луч" проходит по кристаллической решётке, подсвечивая найденные элементы. Они визуально "подпрыгивают" и группируются.
- **Добавить индекс**: Кнопка "CREATE INDEX". При нажатии — видно как в решётке появляется дополнительная структура (B-tree визуально). Повторный запрос проходит значительно быстрее (визуально заметно).
- **Изменить схему**: Drag-and-drop на колонки данных, чтобы перестроить структуру. Решётка перекристаллизовывается с анимацией.
- **Kill query**: Если запрос "завис" — кнопка отмены с визуальной остановкой луча.

**Звук**: текучий ambient при потоке данных, кристаллический "ping" при индексации, глухой "thud" при отклонении невалидных данных.

**Теги**: `PostgreSQL` `Redis` `MongoDB` `SQLAlchemy` `ETL` `Indexing`

### 6.5 ПРОЕКТ 5: "Chain Pulse" — Крипто и блокчейн

**Концепция**: Живая визуализация блокчейна.

**Визуал**:
- Горизонтальная цепочка блоков, растущая слева направо.
- Каждый блок — полупрозрачный куб с данными внутри (hash, nonce, транзакции как светящиеся нити).
- Между блоками — chain-links (визуальные "цепи" или световые мосты).
- Слева — "mempool" (бассейн ожидания): хаотично плавающие транзакции-сферы разного размера (размер = gas fee).
- Фон: тёмный с пульсирующей сеткой (grid), как heartbeat monitor.

**Интерактивность**:
- **"Create Transaction"**: Посетитель вводит от-кого, кому, сумму. Транзакция визуализируется как светящаяся сфера и попадает в mempool. Видно:
  1. Транзакция появляется в бассейне с splash-эффектом
  2. "Валидаторы" (визуально — сканирующие лучи) проверяют транзакцию
  3. Транзакция включается в следующий блок
  4. Блок "кристаллизуется" (частицы собираются в куб) с эффектом свечения
  5. Цепочка удлиняется, камера плавно сдвигается
- **Hover на блок**: Блок "раскрывается" — видны все транзакции внутри, hash, предыдущий hash, timestamp.
- **Переключатель сетей**: Bitcoin / Ethereum / Solana — меняется скорость создания блоков, визуальный стиль, размер блоков.
- **Gas Visualizer**: Ползунок gas price. При изменении — видно как транзакции в mempool перестраиваются по приоритету. Дорогие — поднимаются вверх и попадают в блок первыми.
- **51% Attack** (пасхалка): Скрытая кнопка. При нажатии — альтернативная цепочка начинает расти быстрее, обгоняет основную. Визуальный "battle" двух цепей с драматическим эффектом. Потом сброс.

**Звук**: глубокий "boom" при создании блока, электронный chirp при валидации, heartbeat-пульс у mempool.

**Теги**: `Web3.py` `Ethereum` `Smart Contracts` `Blockchain` `Solana` `Cryptography`

### 7.2 Интерактивный ASCII-арт
Внизу контактной секции — ASCII-арт (имя, лого, или абстрактный паттерн), составленный из символов. При наведении курсора символы "разлетаются" (каждый символ — отдельный DOM-элемент с физикой), создавая волну от курсора. После отведения — собираются обратно.

Реализация: каждый символ — `<span>` с absolute positioning. Физика через requestAnimationFrame: сила отталкивания от курсора, пружинная сила возврата к исходной позиции, затухание.

### 7.3 Magnetic hover на ссылках
Иконки соцсетей и email-блок: при приближении курсора в радиусе ~80px элемент начинает "притягиваться" к курсору (translate в сторону курсора на 10–20px). При выходе — возвращается с пружинным ease. На hover — aurora glow-эффект вокруг элемента.

## 8. КАСТОМНЫЙ КУРСОР

### 8.1 Структура
Два элемента:
1. **Точка** (inner): 8px, aurora-gradient заливка, следует за курсором мгновенно.
2. **Кольцо** (outer): 40px, обводка 1px, следует за курсором с задержкой (lerp 0.15) — создаёт эффект "тянущегося" курсора.

### 8.2 Состояния
- **Default**: Точка + кольцо, нормальный размер.
- **Hover (кликабельное)**: Кольцо увеличивается до 60px, внутри появляется текст "View" / "Drag" / "Click". Mix-blend-mode: difference.
- **Hover (текст)**: Курсор превращается в вертикальную линию (text cursor).
- **Drag (перетаскивание)**: Кольцо сжимается, точка увеличивается, курсор "схватил" объект.
- **Active (клик)**: Мгновенное уменьшение с elastic bounce обратно.

### 8.3 Trail-эффект
За кольцом — лёгкий шлейф из 5–8 "призраков" (copies с уменьшающейся opacity и размером). Каждый призрак следует с увеличивающейся задержкой (lerp 0.1, 0.08, 0.06...). Эффект заметен только при быстром движении.

## 10. ЗАГРУЗКА И ПРОИЗВОДИТЕЛЬНОСТЬ

### 10.1 Loading Screen
При первом заходе — минималистичный loading-экран:
- Чёрный фон.
- По центру — текст "EDWARD" с glitch-эффектом (буквы рандомно перебираются, как в хакерских фильмах).
- Под текстом — progress bar, стилизованный как terminal: `[████████░░░░░░] 67%`
- Загружаются: шрифты, текстуры, шейдеры, звуки. Всё параллельно.
- При завершении загрузки — текст "стабилизируется", progress bar заполняется, затем экран плавно переходит в hero.

### 10.2 Оптимизация
- **Ленивая загрузка сцен**: Только hero загружается сразу. Сцены проектов загружаются на 1 секцию вперёд (prefetch следующей сцены при входе в текущую).
- **Шейдеры**: Компилируются при загрузке, кешируются.
- **Текстуры**: WebP/AVIF, максимальный размер 512×512 для UI-элементов.
- **Three.js**: `antialias: true` на десктопе, `false` на мобильных. `pixelRatio` ограничен до 2.
- **Звуки**: Загружаются только после включения звука пользователем.
- **Target**: First Contentful Paint < 1.5s, Time to Interactive < 3s.
